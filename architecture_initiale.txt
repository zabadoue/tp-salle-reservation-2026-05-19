    @property
    def id_salle(self) -> int: return self._id_salle

### 3. SPECS FRONTEND
**Design :** Utilisation d'une palette "Soft Modern" (Fond: `#f4f7f6`, Cartes: `#ffffff`, Accent: `#3498db`).

*   **`index.html` :**
    *   Conteneur principal `<div class="container">`.
    *   Header avec `<h1>`.
    *   Section `<div id="salles-list">` pour injection dynamique via JS.
*   **`style.css` :**
    *   `body` : Flexbox, centrage total, `min-height: 100vh`.
    *   `.container` : `max-width: 800px`, `padding: 2rem`, `box-shadow: 0 4px 6px rgba(0,0,0,0.1)`, `border-radius: 12px`.
    *   `.salle-card` : Design épuré, bordure légère, bouton d'action avec transition `hover`.

### 4. FLUX DE DONNÉES
1.  **Initialisation :** Le backend expose une API (ex: FastAPI ou Flask) avec des endpoints `/salles` (GET) et `/reserver` (POST).
2.  **Chargement :** Au chargement de `index.html`, un script JS effectue un `fetch()` vers `/salles`.
3.  **Rendu :** Le frontend génère dynamiquement les éléments HTML basés sur le JSON reçu.
4.  **Action :** Lors du clic sur "Réserver", le frontend envoie l'ID de la salle au backend. Le backend met à jour l'attribut `_est_reservee` et renvoie un statut de succès/échec.
5.  **Rafraîchissement :** Le frontend met à jour l'état visuel de la carte correspondante sans recharger la page.
